# 微信投诉分段截图采集工具

这个工具用于在微信电脑端“消费者投诉”页面中，批量采集右侧投诉详情，并按每个投诉输出一个文件夹，里面保存多张分段截图。  
现在还支持在批量采集完成后，自动基于最新一次采集目录生成 Excel 表格。

## 当前功能

- 可视化拖拽框选微信窗口、左侧投诉列表、右侧完整内容区
- 预览当前投诉的分段截图效果
- OCR 识别左侧可见投诉项并自动逐条点击
- 从右侧当前底部位置开始，自动向上逐张截图
- 相邻截图尽量裁掉重复内容
- 左侧自动翻页并继续采集
- 输出 `run_manifest.json` 记录成功项与失败项
- 批量采集完成后，自动按最新一次采集目录生成 Excel 表格
- 支持手动点击“按最新采集结果生成表格”单独补跑

## 运行

```powershell
python .\run_tool.py
```

## 建议使用方式

1. 先把微信打开到“消费者投诉”页面，并固定窗口位置和大小。
2. 打开工具，点击“设置区域”，按顺序完成 3 次框选：
   - 微信窗口
   - 左侧投诉列表
   - 右侧完整内容区
3. 点击“预览当前投诉分段图”，确认输出目录中已经生成多张顺序截图。
4. 预览正常后，点击“开始批量采集”。
5. 批量采集结束后，工具会自动读取最新一次采集目录，并在该目录下生成填写后的 Excel 表格。

## `order_info.png` 说明

- 从当前版本开始，每个投诉文件夹里除了 `001.png`、`002.png` 这类分段截图，还会额外生成一张 `order_info.png`
- 这张图会在右侧内容自动滚到最上方后保存，并且只保留更偏顶部的一段，主要用于识别订单信息卡片
- Excel 导出时会优先使用这张图补抓：
  - `交易单号`
  - `商户单号`
  - `交易时间`
  - `咨询时间`
  - `问题描述`
- `order_info.png` 不会插入到表格的“沟通记录”图片列中
- 如果你想验证新版本是否生效，最直接的方法就是重新采一批，看每个投诉目录里是否已经有 `order_info.png`

## 表格导出规则

- 固定使用 `work/处理表格.xlsx` 作为模板
- 默认处理最新一次采集目录，例如 `image/20260524_155409`
- 输出表格保存在对应采集目录下，例如：

```text
image/20260524_155409/20260524_155409_处理表格.xlsx
```

- 自动填写这些字段：
  - `交易单号`
  - `商户单号`
  - `退款情况`
  - `处理结果`
- 当前规则：
  - 检测到全额退款/已退款到账/原路返回等文案，写入：
    - `退款情况`：`全额退款`
    - `处理结果`：`全额退款（原路返回）`
  - 未检测到退款文案，写入：
    - `退款情况`：`未退款`
    - `处理结果`：`未退款`
- `订单商品说明` 和 `未退款情况说明` 先留空
- `沟通记录` 会插入截图，插入顺序与采集顺序相反
  - 末位序号更大的图片放前面
  - 最后一张采集图片不会插入

## 输出结构

- 每次批量采集会生成一个运行目录，例如 `20260524_155409`
- 每个投诉在该目录下生成一个独立文件夹，例如 `0001_Alice_39.80`
- 每个投诉文件夹内保存：
  - `001.png`
  - `002.png`
  - `003.png`
  - `order_info.png`
- 序号越大，表示越靠上的历史内容
- 同一运行目录下还会生成：
  - `run_manifest.json`
  - `20260524_155409_处理表格.xlsx`
